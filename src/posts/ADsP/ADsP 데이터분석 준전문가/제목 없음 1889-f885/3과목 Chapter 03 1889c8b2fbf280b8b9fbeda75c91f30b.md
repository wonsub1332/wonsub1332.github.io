## Chapter 03 정형 데이터 마이닝

### 086 정형 데이터 마이닝

1. 데이터 마이닝 개념
    
    기업이 보유하고 있는 일일 거래 데이터, 고객 데이터, 상품 데이터 혹은 각종 마케팅 활동에 있어서의 고객 반응 데이터 등과 이외의 외부 데이터를 포함하는 모든 사용가능한 원천 데이터를 기반으로 감춰진 지식, 기대하지 못했던 경향 또는 새로운 규칙 등을 발견하고 이를 실제 비즈니스 의사결정 등에 유용한 정보로 활용하는 일련의 작업
    
2. 데이터 마이닝 5단계 (`순서,데이터 준비와 가공 구분!`)
    1. 목적 정의 
        - 데이터 마이닝 도입 목적을 명확하게 함
    2. 데이터 준비
        - 데이터 정제(Cleaning)를 통해 데이터의 품질 확보까지 포함
        - 필요시 보강 작업을 거쳐 데이터 양을 충분하게 확보
    3. 데이터 가공 (주의!)
        - 목적 변수를 정의하고, 필요한 데이터를 데이터 마이닝 소프트웨어에 적용할 수 있게 가공 및 준비하는 단계
        - 충분한 CPU와 메모리,  디스크 공간 등 개발 환경 구축이 선행
    4. 데이터 마이닝 기법 적용
        - 모델을 목적에 맞게 선택하고 소프트웨어를 사용하는 데 필요한 값 지정
    5. 검증
        - 결과에 대한 검증 시행
3. 대표적 데이터 마이닝 기법
    1. 분류 (Classification) : → 지도 학습
        1. 새롭게 나타난 현상을 검토하여 기존의 분류, 정의된 집합에 배정하는 것
        2. 잘 정의된 분류 기준과 선분류(preclassified)되어진 검증 집합에 의해 완성됨
        
        예 ) 생물을 종, 속, 과로 나누는 것
        
    2. 추정 (Estimation) :
        1. 연속된 변수의 값을 추정
        2. 주어진 입력 데이터이를 사용하여 알려지지 않은 결과의 값을 추정하는 것
        
        예 ) 가족 구성원의 총 수입 추정, 고객의 평생 가치 추정
        
    3. 연관분석 (Association Analysis) :
        1. ‘같이 팔리는 물건’ 같이 아이템의 연관성을 파악하는 분석
        2. 카탈로그 배열 및 교차 판매, 공격적 판촉 행사 등의 마케팅 계획
        
        예)  시장 세분화의 첫 단계로, 판촉 활동에 가장 반응률이 높은 고객을 선별
        
    4. 군집 (Clustering) : → 비지도 학습
        1. 미리 정의된 기준이나 예시에 의해서가 아닌 레코드 자체가 가진 다른 레코드와의 유사성에 의해 그룹화되고 이질성에 의해 세분화 됨
        2. 데이터 마이닝이나 모델링의 준비 단계로서 사용됨
        
        <aside>
        💡
        
        군집과 분류의 차이점:
        
        군집은 선분류 되어있는 기준에 의존하지 않는다는 점
        
        </aside>
        
        예) 고객 특성에 따라 고객의 그룹을 형성해 마케팅 전략을 수립하는 데 활용함
        
    5. 기술 (Description) :
        1. 데이터가 가진 특징 및 의미를 단순하게 설명하는 것
        2. 데이터가 암시하는 바에 대해 설명 및 그에 대한 답을 찾아 낼 수 있어야 함
        3. 사람, 상품에 관한 이해를 증가시키기 위한 것으로 데이터의 특징 및 의미를 표현 및 설명하는 기능 

### 087 모형 평가

1. 클래스 불균형 데이터
    1. 분류 모델에서 일부 범주형의 관측치가 현저히 부족하여 모형이 학습하기 힘든 문제 상태
    2. 클래스의 비율이 한쪽에 치우쳐 있음
    3. 클래스 불균형 상태라면 학습 전 언더샘플링(Undersampling)과 오버샘플링(Oversampling) 기법의 사용을 고려함
        1. 언더샘플링 : 범주별 데이터 개수를 적은 class의 수에 맞추는 것
        2. 오버샘플링 : 범주별 데이터 개수를 많은 class의 수에 맞추는 것
            
            ![image.png](3%EA%B3%BC%EB%AA%A9%20Chapter%2003/image.png)
            
    4. 오버샘플링 방법 적용 시 과적합 및 노이즈 증가, 데이터 분포 왜곡, 계산 비용 증가 등의 문제가 발생할 수 있으므로 주의
    5. 모델의 과적합을 방지하기 위해 교차 검증 및 하이퍼 파라미터 튜닝(=조정)을 신중하게 수행해야 함
2. 홀드 아웃 (Holdout)
    
    <aside>
    💡
    
    ![image.png](3%EA%B3%BC%EB%AA%A9%20Chapter%2003/image%201.png)
    
    - Training Data : 학습용 데이터
    - Test Data : 학습 종료 후 성능 확인용 데이터
    - Validation Data : 학습 중 성능 확인용 데이터 (Overfitting 여부 확인, Early Stopping 등을 위해 사용) , 반복되는 학습 기법에서 사용되며 몇 번의 반복을 하면서 과대적합이 되는지 확인하거나 , 과대적합 발생이 감지되면 빠르게 학습을 멈추기 위해 사용하는 데이터
    </aside>
    
    - 특징
        - 원천 데이터를 랜덤 추출 방법을 활용하여 데이터가 편향되지 않도록 두 개로 분리하여 과대 적합 발생여부와 성능을 평가하는 방법
        - 하나는 모형 학습(=Train)을 위한 자료로 다른 하나는 모형 성과 평가를 위한 평가용 자료로 사용함
        - 과대적합 발생 여부를 확인하기 위해 train, test 데이터에 대해 각각 평가 후 점수를 비교함
            - Train 점수가 Test 점수에 비해 매우 높은 경우 ⇒ 과대적합
            - Train 점수와 Test 점수 모두 낮은 경우 ⇒ 과소적합
            - Train 점수와 Test 점수가 모두 높으면서 비슷한 경우 적절
            - 일반적으로 Train 점수가 Test 점수보다 높게 나타남
        - 학습 단계(Training Phase)에서 학습용 데이터는 다시 학습과 검증(Validation) 데이터로 나누어 사용할 수 있음
        - 즉, 데이터를 학습(train)세트, 검증(validation) 세트, 테스트 세트 세가지로 나누어 사용할 수 있음
        - 학습 데이터와 테스트 데이터는 겹쳐서는 안되며, 검증 데이터와 테스트 데이터는 일치할 수 있음
        - 분류 모형에서 불균형 데이터의 경우 불균형 범주의 비율을 유지하도록 분할하기도 함(층화추출법 사용)
        - 잘못된 가설을 가정하게 되는 제2종 오류의 발생을 방지함
    - 데이터를 2개로 나누는 R코드
        
        ```r
        set.seed(123)
        idx <- sample(2, nrow(iris), replace=TRUE, prob=c(0.7,0.3))
        trainData <- iris[idx==1,]
        testData <- iris[idx==2,]
        ```
        
        - iris 데이터를 7 : 3 비율로 복원추출을 허용하여 나누어 Training 단계에 70%, Testing 단계에 30% 사용하도록 함
        - set.seed(정수)를 사용하면 랜덤 수의 발생을 고정할 수 있어 여러 번 반복 수행 시에도 동일한 데이터 분할이 될 수 있음
    
    <aside>
    💡
    
    층화추출을 사용한 데이터 분할
    데이터에 A,B 두 개 범주가 있을 때 A가 60%, B가 40%하면, A와 B의 비율이 6:4에 맞춰 train에서도 6:4, Test에서도 6:4이 되도록 데이터를 분할하는 것을 의미함
    
    </aside>
    
3. 교차 검증(Cross Validation)
    1. 교차 검증 개요
        1. 데이터가 충분하지 않은 경우 데이터셋을 train, validation, test의 3개로 나누는 방식의 홀드아웃을 사용하면 validation 데이터의 양이 적기 때문에 데이터가 나뉘는 비율이나 데이터 상태에 따라 검증 결과가 많이 달라질 수 있음
        2. 홀드아웃의 단점에 대한 해결책으로 보다 많은 조각으로 나누어 여러번 검증하는 교차 검증을 사용할 수 있음
        3. 교차 검증은 데이터를 무작위로 섞어 동일한 여러 개로 나누게 되므로, 데이터가 골고루 섞이지 않고 한쪽으로 모이는 현상을 보일 수 있는 클래스 불균형 데이터에는 적합하지 않음
        4. 종류
            
            K-Fold 교차 검증, Stratified K-Fold 교차 검증, LOOCV 등
            
    2. K-Fold 교차 검증
        1. 주어진 데이터를 K개로 나누고, K번 반복적 학습 및 성과를 측정하여 그 결과를 평균한 성능으로 평가하는 방법
        2. 10 Fold 교차 검증은 K=10으로 데이터를 동일한 10개 조각으로 나누고 1개를 검증, 9개를 학습데이터로 사용
        3. 총 10번 검증을 실시하고 얻은 결과를 평균 내어 최종 결과로 사용함
        
        ![image.png](3%EA%B3%BC%EB%AA%A9%20Chapter%2003/image%202.png)
        
        <aside>
        💡
        
        K-Fold Cross Validation
        
        1. 전체 데이터를 섞음 (shuffle)
        2. 데이터를 동일 크기의 K개로 분할 (K의 최솟값 :2)
        3. K번째의 하부 집합을 검증용 자료, 나머지 K-1개는 훈련용 자료로 사용하여 K번 반복 측정
        4. K개의 평가 결과를 평균 낸 값을 최종 평가로 사용
        </aside>
        
    3. LOOCV (Leave One Out Cross Validation) 
        1. 데이터 개수를 n이라고 할 때 n번의 학습을 진행하고 그 평가 결과를 평균하여 최종 결과로 사용
        2. 1개 관측값 만을 validation set으로 사용하고 나머지 n-1개를 train set으로 사용하여 n번의 학습을 진행
        3. ***k=n인 경우의 교차 검증***
        4. 학습 후 계산된 n개의 MSE(Mean Squared Error)를 평균하여 최종 MSE를 계산함
        5. 매우 많은 반복 학습으로 시간이 오래 걸림
        6. 주로 데이터가 한정적이고, 새로운 데이터에 대한 정확한 에러 예측을 위해 사용함
        
        ![image.png](3%EA%B3%BC%EB%AA%A9%20Chapter%2003/image%203.png)
        
4. 붓스트랩 (Bootstrap)
    1. 평가를 반복하는 측면에서 교차 검증과 유사하지만, 훈련용 자료를 반복 재선정한다는 점에서 차이가 있는 평가 방법
    2. 붓스트랩은 관측치를 한 번 이상 훈련용 자료로 사용하는 복원추출법에 기반함
    3. 전체 데이터 양이 크지 않을 경우의 모형 평가에 가장 적합
    4. 훈련 데이터를 63.2% 사용하는 0.632 붓스트랩이 있음, 즉 훈련에 사용하지 않운 데이터가 36.8%가 되는 것

### 088 분류 분석의 모형 종류  `설명을 보고 어떤 종류인지`

1. 로지스틱 회귀 분석 (Logistic Regression)`⭐️⭐️회귀 모델이 아님!!!⭐️⭐️` 
    1. 분류 분석에만 사용하는 선형 모형
    2. 이진 분류에 주로 사용되지만 다중 분류에도 사용할 수 있음
    3. 입력 특성의 가중치 합을 로지스틱 함수에 적용하여 확률을 예측함
2. 의사결정 나무 (Decision Tree)
    1. 트리 구조를 사용하여 데이터를 분류 또는 회귀하는 모델
    2. 노드는 특정 기준으로 데이터를 분할하며, 리프 노드에서 예측값을 반환함
    3. 해석이 쉽고 과적합을 조절하기 위한 가지치기가 가능함
3. 앙상블(Ensemble)
    1. 여러 모델을 결합하여 더 강력한 모델을 생성하는 기법
    2. 앙상블의 종류 : Voting, Bagging, Boosting, Random Forest, Stacking 등
4. K-NN (K-Nearest Neighbors)
    1. 데이터 포인트 주변의 가장 가까운 k개 이웃 데이터 포인트를 기반으로 분류 회귀를 수행하는 비모수적 알고리즘
    2. 새로운 데이터 포인트와 가장 가까운 이웃들의 클래스를 기반으로 예측함
5. SVM(Support Vector Machine)
    1. 데이터를 고차원 공간으로 매핑하여 선형 또는 비선형 분류를 수행하는 모형
    2. 최대 마진을 가진 결정 경계를 찾는 것이 주요 목표이며, 이를 통해 일반화 능력이 높은 분류를 가능하게 함
6. 신경망 모형 (ANN)
    1. 인공신경망 구조를 사용하여 복잡한 패턴을 학습하는 모델
    2. 여러 계층의 뉴런으로 구성되며, 숨겨진 계층을 통해 비선형 함수를 모델링 할 수 있음
    3. 딥러닝에서 사용되며 이미지 인식, 자연어 처리 등의 다양한 분야에서 활용됨
7. 나이브베이즈 모형
    1. 베이즈 이론(Bayes’ Theorem)을 기반으로 하며, 주어진 특성(또는 예측 변수)과 클래스 간의 관계를 모델링함
    2. 확률적 분류 및 분류 문제를 해결하기 위한 간단하면서도 효과적인 통계적 분류 모델

지도학습 (Supervised Learning)

### 089 로지스틱회귀 분석

1. 로지스틱회귀 분석
    1. 로지스틱회귀 분석의 이해
        - 종속변수가 범주형인 경우 적용되는 회귀분석 모형
        - 종속변수와 독립변수 간의 관계식을 이용하여 종속변수가 성공/실패, 사망/생존 과 같이 이항변수(0,1)로 되어 있는 이항 분류 및 그 이상의 집단을 분류하고자 할 때 사용되는 분류 기법
        - 일반 선형 회귀는 X 값의 변화에 따른 Y값의 변화를 알아내는 것으로 X가 1 증가할 때 Y는 회귀 계수 만큼 증가하는 것이며, 로지스틱 회귀는 Y가 성공일 확률에 대한 것을 선형 모델로 알아내는 것
    2. 일반 선형 회귀 분석과 로지스틱 회귀 분석
        
        <aside>
        💡
        
        일반 선형 회귀
        
        - 종속 변수 : 연속형 변수
        
        $$
        Y=\beta_0+\beta_1X_1+\beta_2X_2+\cdots+\beta_nX_n
        $$
        
        - 모형 탐색 방법 : 최소자승법 (LSM, 최소제곱법)
        - 모형 검정 : F-test, T-test
        </aside>

        로지스틱 회귀
        
        - 종속 변수 : 이산형 (범주형) 변수
        
        $$
        P(Y=1|X)=\frac{1}{1+e^{-Z}}(Z=\beta_0+\beta_1X_1+\beta_2X_2+\cdots+\beta_nX_n)
        $$
        
        - 모형 탐색 방법 : 최대우도법(MLE), 가중최소자승법
        - 모형 검정 : $\chi^2$  test
        </aside>

### 090 의사결정나무 (Decision Tree) 모형

1. 의사결정나무 모형
    1. 의사결정나무 개요
        1. 의사결정 규칙을 나무 구조로 나타내 전체 자료를 몇 개의 소집단으로 분류하거나 예측을 수행하는 분석 방법
        2. 분석 과정이 직관적이고 이해하기 쉬움
    2. 의사결정나무 모형
        
        ![image.png](3%EA%B3%BC%EB%AA%A9%20Chapter%2003/image%2014.png)
        
2. 의사결정나무 모형 종류와 특징
    1. 종류
        1. 분류나무(Classification Tree) : 목표변수(=종속변수)가 이산형인 의사결정나무
        2. 회귀나무(Regression Tree) : 목표변수가 연속형인 의사결정나무
    2. 특징
        1. 새로운 데이터를 분류하거나 값을 예측하는 것을 목표로 함
        2. 분리 변수 P차원 공간에 대한 현재 분할은 이전 분할에 영향을 받는다
        (노드 분리를 위한 질문 생성은 이전 노드 분리를 위한 질문의 영향을 받음)
        3. 부모마디마다 자식 마디의 순수도가 증가하도록 분류나무를 형성해 나간다(불순도 감소)
    3. 장점
        1. 구조가 단순하여 해석이 용이
        2. 비모수적 모형으로 선형성, 정규성, 등분산성 등의 수학적 가정이 불필요함
        3. 종속변수로 범주형(이산형)과 수치형(연속형) 변수를 모두 사용할 수 있음
        4. 가지치기(Pruning)을 사용하여 비정상적인 잡음 데이터에 대해서도 민감하지 않게 할 수 있음
    4. 단점
        1. 분류 모델의 경우 분류 기준값의 경계선 부근의 자료값은 비연속성을 가지므로 오차가 큼
        2. 각 에측변수의 효과를 파악하기 어려움(로지스틱 회귀는 회귀 계수를 통한 예측변수의 효과 파악이 가능)
        3. 학습 자료에 심취한 학습 하는 경향이 커서, 새로운 자료에 대한 예측이 불안정할 수 있음
        (의사결정나무 모형은 Overfitting 경향이 있음)
3. 의사결정나무의 결정 규칙
    - 의사결정 나무 모델을 구축할 때 분할 규칙을 사용하여 데이터를 나누고, 정지 규칙을 사용하여 트리 생성을 중지하고, 가지치기 규칙을 사용하여 트리를 단순화하여 모델을 효과적으로 관리할 수 있음
    - 이러한 규칙들은 모델의 정확성과 일반화 능력을 균형 있게 유지하는 데 도움을 줌
    - 규칙
        - 분리 기준 (Split Criterion)
            - 새로운 가지를 만드는 기준을 어떻게 정해야 할까?
            - 어떤 특성(변수)를 사용해 데이터를 분할할지를 결정하는 것으로 정보 이득(Information Gain), 지니 지수, 엔트로피 지수 등을 사용함
            - 정보 이득은 분할 후 불확실성의 감소를 나타내며, 지니 지수, 엔트로피는 분할 후 클래스의 불순도를 측정하는 것
            - 불확실성이 낮아지는 방향, 불순도가 낮아지는 방향 → 순수도가 높아지는 방향으로 분리함
                
                ![image.png](3%EA%B3%BC%EB%AA%A9%20Chapter%2003/image%2015.png)
                
        - 정지 규칙 (Stopping Rule)
            - 더 이상 분리가 일어나지 않고 현재의 마디가 최종 마디가 되도록 하는 규칙
            - 트리가 지나치게 깊어지거나 과대적합되는 것을 방지할 수 있음
            - 예
                - 노드에 속한 최소 샘플 수가 일정 개수 미만일 때 더 이상 분할을 수행하지 않도록 함
                - 트리의 최대 깊이가 일정 수준 이상 되면 멈춤
                - 정보이득 또는 지니 지수의 임계값 사용으로 분할 후의 평가 지표가 일정값 미만이 되면 멈춤
                - ‘불순도 감소량’이 아주 작을 때 정지함
        - 가지치기 규칙 (Pruning Rule)
            - 어느 가지를 쳐내야 예측력이 좋은 나무가 될까?
            - 생성된 의사결정나무를 단순화하는 프로세스로 불필요한 분기를 제거하여 모델의 일반화 능력을 향상시키는 것
            - 최종 노드가 너무 많으면 과대적합 가능성이 커지는데 해결하기 위해 사용함
            - 복잡한 트리를 간단하게 만들어 과대적합을 방지하고 데이터에 대한 지나친 적합을 줄임
            - 의사결정나무는 가지치기의 비용함수 (Cost Function)를 최소로 하는 분기를 찾아내도록 학습함
4. 분류용 불순도 측정 지표
    1. 지니 지수 (Gini Index) → `구할수 있어야함`
        
        $$
        \begin{align}
        \text{Gini}(T)
        &= 1-\sum\left(\frac{\text{각 범주별 수}}{\text{전체 수}} \right)^2\\
        &=1-\sum^k_{i=1}P^2_i
        \end{align}
        $$
        
        - 불순도 측정 지표로 그 값이 작을수록 순수도가 높은 것이다
        (순수도가 높다는 것은 오류가 적어서 분류가 잘된다는 것을 의미함)
        - 가장 작은 값을 갖는 예측 변수와 이때의 최적 분리에 의해 자식 마디 형성
        - 예
            - 🔲 🔲 🔳 🔳
                
                → Gini=$1-((2/4)^2+(2/4)^2) = 1-(1/4+1/4)=1/2$ 
                
            - 🔵🔵🔷🔶 원본 데이터
                - 색으로 구분
                    
                    → $1-((3/4^2)+(1/4)^2)=1-5/8=3/8$
                    
                - 모양으로 구분
                    
                    → $1-((2/4)^2+(2/4)^2)=1-4/8=1/2$
                    
    2. 엔트로피 지수(Entropy Measure)
        
        $$
        \text{Entropy}(T)=-\sum^k_{i=1}P_ilog_2P_1
        $$
        
        - 불순도 측정 지표, 가장 작은 값을 갖는 방법 선택
        - 확률이 0.5인 경우 (P=0.5) 불순도가 가장 높은 상태이며, 이떄의 엔트로피=1
        
        ![image.png](3%EA%B3%BC%EB%AA%A9%20Chapter%2003/image%2016.png)
        
    3. 카이제곱 통계량의 유의확률 (p-value)
        - p-value가 가장 작은 값을 갖는 방법 선택
        - 카이제곱 동질성 검정을 사용
        - 귀무가설 : 집단 간 분포가 동일하다 ↔ 대립가설 : 집단 간 분포가 동일하지 않다
        - p-value가 작을 수록 집단 간 분포가 동일하지 않은 이질성이 높은 상태인 것을 의미한다.
        이것은집단 간 분리가 잘 된다는 것으로 이해할 수 있음

### 091 앙상블 (Ensemble) 모형

1. 앙상블 모형 개요
    1. 여러 개의 분류 모형에 의한 결과를 종합하여 분류의 정확도를 높이는 방법
    2. 성능을 분산시키기 때문에 과적합 감소 효과가 있음
    3. 종류
        1. 보팅, 배깅, 부스팅, 랜덤 포레스트, 스태킹 등
    4. 앙상블 기법은 일반적으로 상호 연관성이 낮은 모델을 결합할 때 가장 효과적임
    5. 이를 ‘다양성’이라고 하며, 모델이 서로 다르고 독립적일 때 각 모델은 데이터의 다른 부분 또는 패턴을 잡아내는 경향이 있으며, 이로 인해 결합된 앙상블 모델은 높은 정확도를 달성할 가능성이 더 높아짐
    6. 상호 연관성이 높은 모델을 앙상블하는 경우 각 모델이 유사한 오류를 만들거나 잘못된 판단할 가능성이 높아지고 다양성이 부족하므로 개별 모델보다 성능 향상이 제한될 수 있음

### 093 인공신경망(ANN) 모형

1. 인공신경망 (ANN, Artificial Neural Network)
    1. 인공신경망 개요
        1. 인공신경망을 이용하면, 분류 및 예측을 할 수 있음
        2. 인공신경망은 입력층, 은닉층, 출력층 3개의 층으로 구성되어 있고, 각 층에 뉴런이 여러 개 포함되어 있음
        3. 뉴런은 페셉트론(perceptron)이라고도 함
        4. 학습 : 
            
            → 입력에 대한 올바른 출력이 나오도록 가중치를 조절하는 것
            
            ![image.png](3%EA%B3%BC%EB%AA%A9%20Chapter%2003/image%2023.png)
            
    2. 신경망 모형의 장단점
        1. 장점
            - 변수의 수가 많거나 입출력변수 간에 복잡한 비선형 관계에 유용함
            - 이상치 잡음에 대해서도 민감하지 않음
            - 입력변수와 목적변수가 연속형이나 이산형인 경우 모두 처리 가능함
        2. 단점
            - 결과에 대한 해석이 쉽지 않음
            - 최적의 모형을 도출하는 것이 상대적으로 어려움
            - 모형이 복잡하면 훈련 과정에 시간이 많이 소요됨
            - 데이터를 정규화 하지 않으면 지역해(Local Minimum)에 빠질 위험이 있음
            - Global minimum이 찾으려고 하는 최적해인데 Local minimum을 만나게 되면 그곳이 최적의 해가 있는 위치로 인지하고 학습을 멈추게 될 수 있음
2. 경사하강법 (Gradient Decent)
    1. 개요
        1. 함수 기울기를 낮은 쪽으로 계속 이동시켜 극값에 이를 때까지 반복시키는 것
        2. 제시된 함수의 기울기의 최소값을 찾아내는 머신러닝 알고리즘
        3. 비용함수(Cost function)을 최소화하기 위해 parameter를 반복적으로 조정하는 과정
    2. 과정
        1. 임의의 parameter값으로 시작
        2. 비용함수(Cost Function) 계산:
            
            Cost Function : 모델을 구성하는 가중치 w의 함수, 시작점에서 곡선의 기울기를 통해 계산함
            
        3. Parameter 값 갱신 : 
            
            W = W - Learning rate * 기울기 미분값
            
        
        n 번의 iteration, 최소값을 향해 수렴함, Learning Rate가 적절해야함

        2. 풀고자 하는 문제 종류에 따라 활성화 함수의 선택이 달라짐
        3. 목표 정확도와 학습시간을 고려하여 선택하고 layer별로 다른 활성화 함수를 사용하는 방법으로 혼합 사용도 가능
        4. 문제 결과가 직선을 따르는 경향이 있으면 ‘선형 함수’를 사용
        5. Sigmoid를 사용하는 경우 층이 많아지면 기울기 소실 문제가 발생할 수 있음
    2. 활성화 함수의 종류 (계단, 부호, 선형)

        - Sigmoid 함수
            - 이진 분류 문제에 사용됨
            - 연속형 0~1의 범위를 갖으며, Logistic 함수라고도 함
            - 선형적인 멀티-퍼셉트론에서 비선형 값을 얻기 위해 사용함
        - Softmax 함수
            - 다중 클래스(3개 이상의 범주) 분류 문제에서 주로 사용함
            - 모든 logits의 합이 1이 되도록 output을 정규화함
            - Sigmoid 함수의 일반화된 형태로 결과가 다 범주인 경우 각 범주에 속할 사후 확률을 제공하는 활성화 함수
        - ReLU 함수
            - 음수 입력에 대해 0, 양수 입력에 대해 입력 값을 그대로 출력

        - Leaky ReLU / ELU (Exponential Linear Unit)

            - 함수의 중심점을 0으로 옮겨 sigmoid가 갖고 있던 문제인 최적화 과정에서 느려지는 것을 해결함
            - 연속형 -1 ~ 1의 범위를 갖음
            - 기울시 소실(Vanishing Gradient) 문제는 해결하지 못함

    2. 은닉층 수와 은닉 노드 수의 결정은 ‘분석가가 분석 경험에 의해 설정‘함
    3. 은닉층 노드가 너무 적은 경우
        1. 네트워크가 복잡한 의사결정 경계를 만들 수 없음
        2. 과소 적합(Underfitting) 문제 발생
    4. 너무 많은 경우
        1. 복잡성을 잡아낼 수 있지만, 일반화가 어려움
        2. 레이어가 많아지면 기울기 소실 문제가 발생할 수 있음
        3. 과적합(Overfitting) 문제 발생

    2. 계산된 기울기를 사용하여 각 가중치 매개변수를 업데이트함

    4. Sigmoid 함수를 활성화 함수로 사용할 때 발생, 기울기 소실 문제의 해결을 위해 ReLU 등 다른 함수를 활성화 함수로 사용함

### 094 분류 모형 평기 지표 1 - 오분류표 `계산, 식암기`

1. 오분류표에 사용되는 약어의 의미
    
    ![image.png](3%EA%B3%BC%EB%AA%A9%20Chapter%2003/image%2028.png)
    
    - 오분류표는 실제값과 예측값의 일치, 불일체에 대해 집계한 표이다
        
        
        |  | 결과 | 예측, 실제 상황 |
        | --- | --- | --- |
        | TP | O | 예측 True , 실제 True  |
        | TN | O | 예측 False , 실제 False |
        | FP | X | 예측 True, 실제 False |
        | FN | X | 예측 False, 실제 True |
2. 오분류표 (=혼동행렬, Confusion Matrix)
    - 구분이 예측/실제 또는 실제/예측으로 되어 있거나 FALSE/TRUE, TRUE/FALSE 등으로 나왔을 때에도 정확하게 표를 이해할수 있어야 한다
    - 문제에서 오분류표가 제시되면 ‘실제/예측, TRUE/FALSE의 위치를 먼저 확인‘하고 문제를 풀이하도록 한다
        
        ![image.png](3%EA%B3%BC%EB%AA%A9%20Chapter%2003/image%2029.png)
        
    - 오분류표를 사용하여 Accuracy, Error Rate, Precision, Recall, F1 Score, Specificity, FP Rate와 같은 분류 모형 성능평가 지표를 구할 수 있도록 연습해야 한다 ⭐️⭐️⭐️
        
        ![image.png](3%EA%B3%BC%EB%AA%A9%20Chapter%2003/image%2030.png)
        
        - 실제가 TRUE인 것에 대해 예측도 TRUE인 것이 → Sensitivity
        - 예측이 TRUE인 것에 대해 실제도 TRUE인 것이 → Precision
    - 분류 모형 오분류표
        - Accuracy (정확도, 정분류율)
            - 전체 예측에서 옳은 예측의 비율
            - 식 :
                
                $$
                \text{Accuracy}=\frac{TN+TP}{TN+TP+FN+FP}
                $$
                
        - Error Rate (오분류율)
            - 전체 예측에서 틀린 예측의 비율
            - 식 :
                
                $$
                \text{Error Rate}=\frac{FN+FP}{TN+TP+FN+FP}
                $$
                
        - Precision (정밀도)
            - 예측 값이 True 인 것에 대해 실제 값이 True인 지표
            - 식 :
                
                $$
                \text{Precision}=\frac{TP}{TP+FP}
                $$
                
        - Recall, Sensitivity (재현율, 민감도)
            - 실제 값이 True인 것에 대해 예측 값이 True인 지표
            - 식 :
                
                $$
                \text{Recall , Sensitivity}=\frac{TP}{TP+FN}
                $$
                
        - F1 Score
            - 불균형한 데이터(Imbalanced Data) 평가에 사용함
            - 정밀도와 재현율의 조화평균, 정밀도와 재현율에 같은 가중치를 부여하여 평균한 지표
            - 식 :
                
                $$
                F1=2\times \frac{\text{Precision}\times \text{Recall}}{\text{Precision}+ \text{Recall}}
                $$
                
            - Accuracy의 경우 불균형한 레이블 값 분포의 데이터에서는 모델의 성능이 실제로 좋지 못하더라도 정확도기 높을 수 있음
        - F2 Score
            - F 뒤의 숫자는 재현율에 부여하는 가중치를 주는 방식을 의미함
            - 재현율에 정밀도의 2배만큼 가중치를 부여하는 것
            - 식 :
                
                $$
                F_\beta=(1+\beta^2)\times\frac{\text{Precision}\times \text{Recall}}{(\beta^2\times\text{Precision})+\text{Recall}}
                $$
                
            - 예 :
                
                $$
                F_2=(1+2^2)\times\frac{\text{Precision}\times \text{Recall}}{(2^2\times\text{Precision})+\text{Recall}}
                $$
                
        - Specificity (특이도)
            - 일반적인 지표들은 True로 맞춘 것에 대해 관심이 있는데, 특이도는 False로 맞춘 것에 대해 관심이 있음
            - 실제 False인 것 중에서 예측도 False여서 맞춘것의 비율
            - 식 :
                
                $$
                \text{Specificity}=\frac{TN}{TN+FP}
                $$
                
        - FP Rate (1종 오류)
            - 실제 False인 것 중에서 예측이 True로 틀린 것의 비율
                - Y가 아닌데 Y로 예측된 비율, 1종 오류 비율
            - 식
                
                $$
                \text{FP Rate}=\frac{FP}{TN+FP}=1-\text{Specificity(특이도)}
                $$

### 095 분류 모형 평가 지표 2

1. ROC (Receiver Operating Characteristic) 그래프
    1. X축은 FP Rate(1-specificity), Y축은 민감도(sensitivity)를 나타내 이 두 평가 값의 관계로 모형을 평가함
    2. ROC 그래프의 밑부분의 면적 (AUC, Area Under the Curve)이 넓을수록 좋은 모형으로 평가함
    3. AUC 값은 0~1사이의 값을 갖으며, 그 값이 1에 가까울수록 더 넓은 것이며 좋콟

    - 실제가 False인데 예측이 True로된 비율 (1종 오류 비율) $\text{FP-Rate}=\frac{FP}{FP+TN}$

2. 카파 상관계수 (Kappa)
    
    $$
    K=\frac{\text{Accuracy}-P(e)}{1-P(e)}\\
    \#\ P(e):\text{우연히 일치할 확률, Accuracy를 P(o)또는 P(a)로 표기하기도함 }
    $$
    
    - 코헨의 상관계수로 두 평가자의 평가가 얼마나 일치하는지 평가하는 값
    - 0~1 사이의 값을 가지며, 1에 가까울수록 일치 정도가 높다고 판단함, P(e)가 클수록 일치정도 높음
    - 모델 예측값과 실제값의 일치 여부를 판단하는 통계량으로 사용됨
3. 향상도 도표 (Lift Table)
    - 분류 모형의 예측 성능을 평가하기 위한 척도, 주로 불균형 데이터 집합에 사용됨

    - 임의로 나눈 각 등급(decile)별로 반응 검출률, 반응률, Gain(이득), Lift(향상도) 등의 정보를 산출하여 나타내는 표

2. 계층적 군집 (Hierarchical Clustering)의 특징
    1. 가장 유사한 개체를 묶어 나가는 과정을 반복하여 원하는 개수의 군집을 형성하는 방법
    2. 유사도 판단은 두 개체 간의 거리에 기반하므로 거리 측정에 대한 정의가 필요함
        1. 유사도, 맨해튼, 민코프스키, 마할라노비스 등
    3. 이상치에 민감함 (거리에 기반하는 경우 이상치에 민감한 특징을 갖게 됨)
    4. 사전에 군집 수 k를 설정할 필요가 없는 탐색적 모형
    5. 군집을 형성하는 데 매 단계에서 지역적 최적화를 수행해 나가는 방법을 사용하므로 그 결과가 전역적인 최적해라고 볼 수 없음
    6. 병합적 방법에서 한 번 군집이 형성되면 군집에 속한 개체는 다른 군집으로 이동할 수 없음
    7. hclust() 함수, cluster 패키지의 agnes(), mclust() 함수 사용
3. 계층적 군집의 거리
    1. 수학적 거리 개념
        1. 유클리드 (Euclidean)
            - 두 점 사이의 직선 거리를 말하며, 가장 직관적이고 일반적인 거리의 개념
            - 두 점의 성분별 차의 제곱의 합에 대한 제곱근
        2. 맨해튼(Manhattan)
            - 두 점의 성분별 차의 절댓값의 함
        3. 민코프스키(Minkowski)
            - 거리 차수($p$)에 따라 유클리드 또는 맨해튼 거리 개념으로 사용됨
            - 거리 차수 $p=1$ 이면 맨해튼 거리, $p=2$ 이면 유클리드 거리가 됨
    2. 통계적 거리 개념
        1. 표준화 거리 (Standardized Distance)
            - 각 변수를 해당 변수의 표준편차로 척도 변환 후, 유클리드 거리를 계산한 것
            - 표준화를 하면 척도의 차이, 분산의 차이로 인한 왜곡을 피할 수 있음
            
            $$
            \sqrt{\sum_{i=1}^n\left(\frac{x_i-y_i}{s_i} \right)^2}
            $$
            
        2. 마할라노비스 (Mahalanobis Distance)
            - 변수의 표준화와 함께 변수 간의 상관성을 동시에 고려한 통계적 거리
    3. R에서의 거리 함수 & 코사인 거리
        1. dist 함수
            - R에서 거리 측정에 사용되는 함수
            - 사용 가능한 거리 개념 : 유클리드, 맨해튼, 민코프스키, maximum, Canberra, binary 등
        2. 코사인 거리
            - 두 벡터 사이의 사잇각을 계산해서 유사한 정도를 구하는 것
            - 코사인 거리 = 1 - 코사인 유사도 ( Cosine distance = 1- cosine similarity)
            - 값이 1인 경우 유사도가 크며, 값이 -1인 경우 유사도가 매우 작음을 의미함

4. 계층적 군집 - 응집형(병합 군집) 군집의 종류
    1. 최단 연결법 (Single Linkage Method, 단일 연결법) :
        - 두 군집 사이의 거리를 군집에서 하니씩 관측 값을 뽑았을 때 나타날 수 있는 거리의 최솟값을 측정
        - 고립된 군집을 찾는데 중점을 둔 방식
    2. 최장 연결법 (Complete Linkage Method, 완전 연결법) :
        - 두 군집 사이의 거리를 군집에서 하나씩 관측 값을 뽑았을 때 나타날 수 있는 거리의 최댓값을 측정
    3. 중심 연결법 (Centroid Linkage Method) :
        - 두 군집의 중심 간의 거리를 측정함
        - 두 군집이 결합될 때 새로운 군집의 평균은 가중 평균을 통해 구해 짐
    4. 평균 연결법 (Average Linkage Method) :
        - 모든 항목에 대한 거리 평균을 구하면서 군집화, 계산양이 많아질 수 있음
    5. 와드 연결법 (Ward Linkage Method) :
        - 계층적 군집내의 오차제곱합에 기초하여 군집을 수행하는 군집 방법, 크기가 비슷한 군집끼리 병합하는 경향이 있음

### 099 군집 분석 - 평가, 결과 해석

1. 군집 분석 평가 지표
    1. 실루엣 계수 (Silhouette Coefficient)
        1. 군집내 거리와 군집 간의 거리를 기준으로 군집 분할 성과를 측정하는 방식
        2. 클러스터(Cluster)안의 데이터들이 다른 클러스터와 비교해 얼마나 비슷한가를 나타내는 군집 평가 지표
        3. 실루엣 지표가 1에 가까울수록 군집화가 잘 되었다고 판단함
        4. 실루엣 지표가 1에 가까울수록 군집화가 잘 되었다고 판단함
        5. 실루엣 지표가 1 : 군집 내 모든 개체가 한치도 떨어져 있지 않고 붙어있는 경우이며, 군집 간 완벽히 분리된 경우임
        6. 실루엣 지표가 0.5 보다 크면 결과가 타당한 것으로 평가함
    2. Dunn Index(DI)
        
        $$
        DI=\frac{\text{군집과 군집 사이 거리 중 최솟값}}{\text{군집 내 데이터들 거리 중 최댓값}}
        $$
        
        - 분자가 클 수록 군집 간 거리가 멀고, 분모가 작을 수록 군집 내 데이터가 모여 있음을 의미함
        - Dunn Index가 클수록 군집화가 잘 되었다고 평가
2. 군집 개수 구하기
    1. 실루엣 스코어를 사용한 군집 수

        BIC (Baysian Information Criterion)
        
        - BIC는 모델의 적합도와 복잡성을 고려하는 통계적 기준이다
        - R언어의 mclust 패키지를 사용한 결과에서 BIC가 가장 큰값인 모형을 선택하고, 그때의 X축 값이 최적의 군집수가 됨
        - 적합도 :
            - 모델이 데이터를 얼마나 잘 설명하는지를 나타내며, 로그-우도(Log-Linklihood)에 의해 평가된다. 높을 수록 좋은 모델을 의미함
        - 복잡성 :
            - 모델의 파라미터 수에 대한 패널티, 더 많은 파라미터를 가진 모델에 더 큰 패널티가 부여되어 더 복잡한 모델일수록 BIC가 더 높아짐

3. 군집 분석의 안정성, 타당성
    1. 군집 분석의 안정성과 타당성은 모두 중요한 요소임
        1. 두가지를 모두 고려하여 최적의 결과를 도출하여야 함 (분석의 목적과 우선 순위에 따라 달라질 수 있음)
        2. 안정성, 타당성을 점검하기는 어려움
    2. 안정성 : 군집 분석의 결과가 변동 없이 일관되게 나타나는지 나타냄
        1. 안정성이 높을수록 동일 데이터에 대해 반복 수행 시 결과가 일관되어 신뢰성이 높음
        2. 집단별 특성이 유사할 경우 안정성이 높을 수 있지만, 데이터 복합성, 다양성에 따라 결과가 달라질 수 있음
    3. 타당성 : 군집 분석 결과가 실제 데이터의 특성과 부합하는지 나타냄
        1. 타당성이 높을수록 군집 분석 결과가 실제로 의미 있는 구조를 잘 나타내며, 결과를 신뢰할 수 있음
    4. 군집 결과에 대한 타당성과 안정성에 대한 검정으로 교차타당성(cross-validation)을 이용할 수 있음
        1. 데이터를 A,B 두개 부분으로 랜덤하게 분류해 놓은 다음 각 부분에서 따로 군집분석을 한 후, 합쳐서 군집 분석한 결과와 비교하여 비슷하면 결과에 대한 안정성이 있다고 판단함
        2. 지도 학습의 교차타당성과 동일한 방법은 아님(지도학습은 결과를 평균!)

### 100 군집 분석 - 자기조직화지도(SOM)

1. SOM (Self-Organizing Map)
    1. SOM의 개요
        1. 자기조직화지도, Kohonen Neural Network (코호넨 신경망)이라고도 함
        2. 비지도 학습의 한가지 방법
        3. 인공 신경망의 한 종류로, 차원축소와 군집화를 동시에 수행하는 기법
        4. 고차원으로 표현된 데이터를 저차원으로 변환해서 보는데 유용함
        5. 입력층과 2차원의 격자 형태의 경쟁층으로 이루어져 있음 (2개의 층으로 구성)
        
        ![image.png](3%EA%B3%BC%EB%AA%A9%20Chapter%2003/image%2054.png)
        
    2. 입력층과 경쟁층
        1. 입력층 (Input Layer) :
            - 입력 벡터를 받는 층으로 입력 변수의 개수와 동일한 뉴런 수가 존재함
            - 입력층에 있는 각각의 뉴런은 경쟁층에 있는 각각의 뉴런들과 완전 연결(Fully Connected)되어 있음
        2. 경쟁층 (Competitive Layer) :
            - 입력 벡터의 특성에 따라 벡터가 한 점으로 클러스터링 되는 층
            - 입력 데이터와의 유사성을 경쟁하여, 각 노드 또는 뉴런들 사이의 경쟁적 상호 작용을 담당함
            - 승자만이 출력을 내고, 승자와 그의 이웃만이 연결 강도를 수정하는 승자 독점
            - 구조로 인해 경쟁층에는 입력 데이터와 가장 유사한 뉴런인 승자 뉴런만 나타남
2. SOM 과정
    1. SOM Process
        1. 단계1 : SOM의 노드에 대한 연결강도(weight) 초기화
        2. 단계2 : 입력 벡터와 경쟁 층 노트 간의 거리 계산 및 입력벡터와 가까운 노드 선택 → 경쟁
        3. 단계3 : 경쟁에서 선택된 노드와 이웃 노드의 가중치(연결강도) 갱신 → 협력 및 적응
        4. 단계4 : 단계2로 가서 반복
    2. SOM의 결과
        1. 승자 독점 구조로 인해 경쟁층에는 승자 뉴런만 나타남
        2. 경쟁학습으로 각각의 뉴런이 입력 벡터와 얼마나 가까운가를 계산하여 연결 강도를 반복적으로 재조정하여 학습함 이와 같은 과정을 거치면서 입력 패턴과 가장 유사한 경쟁층 뉴런이 승자가 됨
        3. 입력 변수의 위치 관계를 그대로 보존한다는 특징으로 인해 입력 변수의 정보와 그들의 관계가 지도상에 그대로 나타남
        4. 고차원의 데이터를 저차원의 지도 형태로 형상화하기 때문에 시각적으로 이해하기 쉬울 뿐 아니라 입력 변수의 위치 관계를 그대로 보존하기 때문에 실제 데이터가 유사하면 지도상 가깝게 표현됨

    1. 연관 분석의 개요
        1. 연관규칙 : 항목들 간의 ‘조건-결과’ 식으로 표현되는 유용한 패턴

        4. 교차 판매, 물건 배치 등에 이용
    2. 연관분석 장점
        1. 조건반응(if-then)으로 표현되는 연관 분석의 결과를 이해하기 쉬움

    3. 연관분석 단점
        1. 분석 품목수가 증가하면 분석 계산이 기하급수적으로 증가함
        2. 너무 세분화된 품목을 가지고 연관 규칙을 찾으려면 의미 없는 분석 결과가 도출됨
        3. 상대적 거래량이 적으면 규칙 발견 시 제외되기 쉬움
    
    <aside>
    💡
    
    비목적성 분석 기법
    
    - 명시적인 목표나 레이블 없이 데이터의 구조를 파악하거나 해석하기 위해 사용됨
    - 주로 군집화(Clustering), 차원 축소(Dimensionality Reduction), 이상치 탐지(Outlier Detection), 연관 규칙 학습(Association Rule Learning)등의 기법을 포함함

2. 연관분석 알고리즘
    1. Apriori 알고리즘
        1. 연관규칙의 대표적 알고리즘으로 현재도 많이 사용됨
        2. 데이터들에 대한 발생빈도를 기반으로 각 데이터 간의 연관 관계를 밝히는 방법
        3. 빈발 항목 집합을 찾는데 기초하며, 지지도 (Support), 신뢰도(Confidence), 향상도(Lift)와 같은 지표를 사용하여 항목 집합의 연관성을 평가함
        4. 모든 가능한 후보 항목 집합을 생성하고 검사하는 과정이 비효율적일 수 있음
        5. 데이터셋이 큰 경우 모든 후보 itemset에 대해 하나하나 검사하는 것이 비효율적임
    2. FP Growth(Frequent Pattern Growth)
        1. Apriori 단점을 보완하기 위해 제안된 알고리즘으로, 구조화된 트리(트리구조를 이용한 데이터 구조)를 사용하여 빈발 항목 집합을 효율적으로 추출함
        2. FP-tree와 node, link라는 특별한 자료 구조를 사용함
        3. 대규모 데이터셋에서의 성능이 우수하며, 더 높은 속도로 빈발 항목 집합을 찾을 수 있음
3. 연관 규칙 측정지표 ⭐⭐⭐
    
    <aside>
    💡 규칙 표기 : A → B ; if A then B가 A가 팔리면 B가 같이 팔린다
    
    </aside>
    
    1. 지지도(Support)
        1. 전체 거리중 차지하는 비율을 통해 해당 연관 규칙이 얼마나 의미가 있는 것인지를 확인함
        2. 전체 거래항목 중 상품 A와 상품 B를 동시에 포함하여 거래하는 비율
        
        $$
        지지도 = P(A\cap B) :\frac{\text{A와 B가 동시에 포함된 거래 수}}{\text{A가 포함된 거래 수}},\\. \\  B\rightarrow A와\ A\rightarrow B\text{의 지지도가 같음}
        $$
        
    2. 신뢰도(Confidence)
        1. 상품 A를 구매했을 때 상품 B를 구매할 확률이 어느 정도 되는지를 확인
        2. 상품 A를 포함하는 거래 주 A와 B가 동시에 거래되는 비율
        
        $$
        신뢰도 =P(B|A)=\frac{P(A\cap B)}{P(A)}=\frac{\text{A와 B가 동시에 포함된 거래 수 }}{\text{A가 포함된 거래 수}},\\.\\ B\rightarrow A\text{의 신뢰도}=P(A|B)=\frac{P(A\cap B)}{P(B)}
        $$
        
    3. 향상도(Lift)
        1. A가 주어지지 않았을 때 B의 확률 대비 A가 주어졌을 때 B의 확률 증가 비율
        2. 품목 B를 구매한 고객 대비 품목 A를 구매한 후 품목 B를 구매하는 고객에 대한 확률
        
        $$
        향상도 =\frac{P(A\cap B)}{P(A)*P(B)},\ B\rightarrow A와\ A\rightarrow B\text{의 향상도 같음}\\=\frac{\text{상품 A의 거래 중 상품 B가 포함된 거래의 비율}}{\text{전체 상품 거래 중 상품 B가 거래된 비율}}=\\.\\ \frac{\text{A와 B가 동시에 일어난 확률}}{\text{A,B가 독립된 사건일 때, A,B가 동시에 일어날 확률}}
        $$

3. 딥러닝
    1. 딥러닝 - 머신 러닝의 한 종류로 인공신경망(ANN)에 기반을 둔 학습 방법
    2. 머신 러닝은 정형 데이터 학습에서, 딥러닝은 이미지, 소리, 텍스트와 같은 비정형 데이터 학습에서 주로 사용됨
    3. 딥러닝 학습을 위한 오픈 소스에는 Tensorflow, Caffe, Theone, PyTorch, Keras등이 있다
    4. 딥러닝 학습의 종류에는 DNN, CNN, RNN, GAN, Autoencoder, 전이학습(Transfer Learning), 강화화습 등이 있다
    - 종류
        - DNN :
            - Deep Neural Network, 일반적인 신경망의 확장판으로 다층으로 쌓인 은닉층을 통해 복잡한 특징을 학습하는 데 사용함
        - CNN :
            - Convolutional NN, 이미지의 공간적인 구조를 보존하면서 특징을 추출하는 데 특화되어 이미지 학습에 주로 사용함
        - RNN :
            - Recurrent NN, 순환 구조를 사용하여 이전 상태 정보를 현재 상태에 전달하여 순차 데이터의 패턴을 학습하고 예측하는 데 효과적인 학습으로, LSTM, GRU 모델이 있음
        - GAN :
            - Generative Adversarial Network, 생성 모델로, 생성자(Generator)와 판별자(Discriminator)로 이루어져 있으며 생성자는 실제 데이터와 유사한 가까 데이터 생성에 판별자는 생성자가 만든 가까 데이터와 실제 데이터 구별에 사용
        - Autoencoder :
            - 비지도 학습 방법으로, 입력 데이터를 저차원의 표현으로 압축한 뒤 다시 복원하는 신경망 구조로 데이터의 효율적인 표현을 학습하고 잡음 제거, 차원 축소 등에 사용된다
        - 전이 학습 :
            - 풍부한 데이터를 이용해서 학습된 Pre-trained Model을 가져와 부족한 사용자 데이터 환경에 맞도록 새롭게 모델을 학습시키는 일련의 과정
